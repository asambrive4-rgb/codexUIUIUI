# Codex Account Switcher — Product MVP

> 문서 상태: Draft  
> 대상 플랫폼: Windows  
> 제품 유형: 개인용 로컬 데스크톱 앱

## 1. 제품 한 줄 정의

> ChatGPT 웹 작업을 끊지 않고, Windows용 Codex에서 사용할 계정을 빠르고 안전하게 전환하는 앱.

## 2. 해결하려는 문제

여러 계정을 사용하는 사람은 Codex 계정을 바꿀 때마다 로그아웃과 로그인을 반복해야 한다. 특히 계정별 사용 한도에 도달했을 때 다른 계정으로 전환하더라도 기존 Codex 작업을 그대로 이어가고 싶다. 이 과정에서 브라우저의 ChatGPT 로그인이나 열어 둔 채팅까지 영향을 받을 수 있고, 현재 어떤 계정을 사용 중인지 파악하기도 어렵다.

사용자는 계정 전환을 위해 Codex를 직접 종료하거나 인증 관련 파일을 다루고 싶지 않다. 원하는 계정을 선택하는 것만으로 전환을 끝내고 싶다.

## 3. 대상 사용자

- Windows용 Codex를 사용하는 개인 사용자
- 개인 계정을 여러개 두고 번갈아 사용하는 사용자
- Codex와 ChatGPT 웹을 동시에 사용하는 사용자

## 4. 제품 목표

- ChatGPT 웹 로그인과 채팅 상태를 유지하면서 Codex 계정만 전환한다.
- Codex의 작업 데이터와 설정은 공통으로 유지하고 계정 인증 상태만 전환한다.
- 여러 Codex 계정을 이름이 있는 프로필로 관리한다.
- 일반적인 실행과 전환을 1~2번의 클릭으로 끝낸다.
- 현재 사용 중인 프로필을 한눈에 보여준다.
- 계정 전환에 필요한 종료와 재실행을 사용자가 직접 처리하지 않게 한다.
- 로그인 정보와 프로필 데이터는 외부 서버로 보내지 않는다.

## 5. 핵심 UX 원칙

| 원칙 | 사용자 경험 |
|---|---|
| 단순함 | 사용자는 프로필을 고르고 `실행` 또는 `전환`만 누른다. |
| 상태의 명확성 | 현재 활성 프로필과 진행 상태를 즉시 알 수 있다. |
| 웹 작업 보호 | 브라우저의 ChatGPT 계정과 열어 둔 채팅은 유지된다. |
| 안전한 전환 | Codex 종료나 재실행이 필요하면 앱이 안내하고 처리한다. |
| 예측 가능한 결과 | 성공, 실패, 추가 확인이 필요한 상태를 명확히 구분한다. |
| 안전한 삭제 | 선택한 프로필의 저장된 인증 정보만 삭제되고 공통 작업 데이터는 유지됨을 확정 전에 알린다. |

## 6. 핵심 사용자 흐름

### 6.1 첫 프로필 추가

1. 사용자가 `새 프로필 추가`를 누른다.
2. 알아보기 쉬운 프로필 이름을 입력한다.
3. `생성 및 로그인`을 누른다.
4. 기존 Codex 로그인 화면에서 로그인한다.
5. 로그인이 끝나면 프로필이 목록에 추가되고 사용할 수 있는 상태로 표시된다.
6. 브라우저의 기존 ChatGPT 로그인과 채팅은 그대로 유지된다.

### 6.2 등록된 프로필로 실행

1. 사용자가 원하는 프로필을 찾는다.
2. Codex가 꺼져 있으면 `실행`을 누른다.
3. 앱은 선택한 프로필로 Codex를 열고 결과를 표시한다.

### 6.3 다른 프로필로 전환

1. 사용자가 다른 프로필의 `전환`을 누른다.
2. 앱은 전환 중임을 표시한다.
3. 필요한 종료와 재실행을 앱이 처리한다.
4. 완료되면 새 활성 프로필을 표시한다.

사용자에게는 이 전체 과정이 하나의 `전환` 동작으로 보여야 한다.

### 6.4 프로필 삭제

1. 사용자가 프로필 메뉴에서 `삭제`를 선택한다.
2. 앱은 저장된 해당 계정의 로그인 정보가 삭제되어 되돌릴 수 없고 공통 작업 데이터는 유지된다고 알린다.
3. 사용자가 확정하면 프로필을 삭제한다.
4. 삭제가 끝나면 목록에서 제거한다.

활성 프로필을 바로 삭제할 수 없다면, 필요한 조치와 이유를 먼저 안내한다.

## 7. MVP 화면 범위

### 7.1 메인 화면

- 프로필 목록
- 현재 활성 프로필 표시
- 프로필별 `실행` 또는 `전환` 버튼
- 프로필별 관리 메뉴
- `새 프로필 추가` 버튼
- 진행 상태와 오류 메시지

프로필에는 최소한 이름과 현재 상태를 표시한다. 마지막 사용 시각은 화면을 복잡하게 만들지 않는 범위에서 제공할 수 있다.

### 7.2 새 프로필 추가

- 프로필 이름 입력
- 입력 오류 안내
- `취소`
- `생성 및 로그인`

빈 이름과 중복 이름은 사용할 수 없다. 이메일, 비밀번호, 인증 토큰을 이 화면에서 직접 입력받지 않는다.

### 7.3 삭제 확인

권장 문구:

> **‘Work’ 프로필을 삭제하시겠습니까?**  
> 이 프로필에 저장된 Codex 로그인 정보가 삭제됩니다. 공통 Codex 작업 데이터와 설정은 삭제되지 않습니다. 로그인 정보 삭제는 되돌릴 수 없습니다.

- `취소`
- `프로필 및 로그인 정보 삭제`

### 7.4 종료 실패 안내

Codex가 정상적으로 종료되지 않으면 작업 손실 가능성을 알리고 다음 선택을 제공한다.

- `취소`
- `강제 종료 후 전환`

## 8. 상태와 피드백

앱은 최소한 다음 상태를 구분해 보여준다.

| 상태 | 표시 원칙 |
|---|---|
| 준비됨 | 실행하거나 전환할 수 있음을 보여준다. |
| 로그인 필요 | 다음 행동이 로그인임을 알려준다. |
| 실행 중 | 현재 활성 프로필을 강조한다. |
| 전환 중 | 중복 조작을 막고 진행 중임을 짧게 표시한다. |
| 확인 필요 | 작업 손실이나 삭제처럼 사용자 결정이 필요한 이유를 설명한다. |
| 실패 | 실패 원인과 사용자가 할 수 있는 다음 행동을 함께 보여준다. |

실패한 작업을 성공으로 표시하거나 활성 프로필을 성급하게 변경해서는 안 된다.

## 9. MVP 포함 범위

- 프로필 목록 보기
- 프로필 생성과 최초 로그인
- Codex 설치 경로 자동 탐지
- 선택한 프로필로 Codex 실행
- 실행 중인 Codex를 다른 프로필로 전환
- 전환 후에도 공통 Codex 작업 데이터와 설정 유지
- 현재 활성 프로필 표시
- 프로필 삭제와 삭제 전 확인
- 실행·전환·삭제 과정의 진행 상태 및 오류 안내
- ChatGPT 웹 세션이 유지되는지 확인

## 10. MVP 제외 범위

- ChatGPT 웹사이트의 계정 전환
- 여러 Codex 계정 동시 실행
- 여러 Windows 사용자 간 프로필 공유
- 조직 또는 관리자 기능
- 외부 서버를 통한 계정 관리
- 이메일·비밀번호·인증 토큰 직접 수집
- 자체 로그인 또는 OAuth 기능
- 클라우드 저장, 동기화 또는 백업
- 프로필 간 대화 기록 병합
- macOS와 Linux 지원
- 자동 업데이트
- 삭제한 프로필 복구

## 11. 품질 및 안전 요구사항

### 사용성

- 일반적인 실행과 전환은 1~2번의 클릭으로 완료한다.
- 프로필 이름과 현재 상태를 한눈에 확인할 수 있어야 한다.
- 사용자가 프로세스, 설정값, 인증 파일을 직접 다루지 않아야 한다.

### 안정성

- 전환이 실패해도 기존 프로필을 손상시키지 않아야 한다.
- 전환 전후에 공통 Codex 작업 데이터와 설정이 그대로 유지되어야 한다.
- 서로 다른 프로필이 동시에 실행되지 않아야 한다.
- 전환 중에는 중복 실행을 막아야 한다.

### 개인정보와 보안

- 프로필과 로그인 관련 데이터는 사용자의 PC에만 보관한다.
- 앱은 이메일, 비밀번호, 인증 토큰을 직접 수집하지 않는다.
- 로그인 정보나 민감한 데이터가 화면의 오류 메시지나 로그에 노출되지 않아야 한다.
- 프로필 삭제 전에는 삭제 범위와 복구 불가능 여부를 분명히 알린다.
- 프로필 삭제가 공통 Codex 작업 데이터나 다른 프로필의 인증 정보에 영향을 주지 않아야 한다.

### 반응성

- 앱을 열면 프로필 목록을 기다림 없이 확인할 수 있어야 한다.
- 실행과 전환 중에는 앱이 멈춘 것처럼 보이지 않도록 진행 상태를 표시한다.

## 12. 성공 기준

MVP는 다음 조건을 모두 만족할 때 성공으로 본다.

- 두 개 이상의 프로필을 만들고 각각 로그인할 수 있다.
- 최초 로그인 후 반복 전환할 때 매번 다시 로그인하지 않아도 된다.
- 프로필 실행과 전환을 1~2번의 클릭으로 완료할 수 있다.
- 계정 전환 후에도 기존 Codex 작업 내역과 설정을 그대로 사용할 수 있다.
- 전환 전후에 ChatGPT 웹 로그인과 열어 둔 채팅이 유지된다.
- 사용자가 Codex를 직접 종료하거나 다시 실행하지 않아도 된다.
- 현재 활성 프로필을 즉시 식별할 수 있다.
- 오류가 발생하면 다음 행동을 이해할 수 있다.
- 서로 다른 프로필이 동시에 실행되지 않는다.
- 프로필 삭제 시 저장된 해당 계정의 로그인 관련 데이터만 삭제되고 공통 작업 데이터는 유지된다.

ChatGPT 웹 세션이 변경되거나 계정 전환 때문에 기존 웹 작업이 끊긴다면 MVP의 핵심 조건을 충족하지 못한 것으로 본다.

## 15. 구현 계획

이 계획은 기능 코드를 한꺼번에 만드는 대신, 가장 위험한 가정부터 검증하고 각 단계의 완료 조건을 통과한 뒤 다음 단계로 진행한다.

### 15.1 구현 전 확인할 사항과 기본 가정

다음 항목은 구현 방식에 큰 영향을 주므로 0단계에서 실제 환경을 조사해 확정한다.

1. Codex의 계정 인증 정보와 공통 작업 데이터가 저장되는 위치 및 서로의 경계
2. Codex 실행 파일과 실행 중인 프로세스를 찾는 방법
3. Codex를 정상 종료할 수 있는 방법과 정상 종료 실패를 판단할 대기 시간
4. 계정 인증 정보만 안전하게 백업·교체·복구하는 방법
5. 지원하는 Codex 설치 유형별 실행 파일 자동 탐지 방법

다음 제품·기술 범위는 확정한다.

- Windows 10/11 x64와 .NET 10 WPF를 지원한다.
- 모든 프로필은 동일한 Codex 로컬 데이터 폴더와 작업 상태를 공유한다.
- 프로필은 계정별 인증 상태의 로컬 사본이며, 전환할 때 인증 관련 항목만 교체한다.
- 앱은 이메일, 비밀번호, 인증 토큰을 직접 입력받거나 인증 데이터의 내용을 해석·표시·기록하지 않는다.
- Codex 실행 파일은 자동 탐지하며, 찾지 못하면 탐지한 위치와 실패 원인을 민감하지 않은 범위에서 안내한다.
- 활성 프로필은 저장된 표시값만 믿지 않고 실제 Codex 프로세스와 현재 적용된 인증 상태를 함께 확인해 판단한다.
- 일반적인 프로세스 실행, 파일 저장, JSON 처리는 .NET 기본 기능을 우선 사용하고 새 라이브러리는 필요한 경우에만 추가한다.

### 15.2 0단계: 핵심 기술 가능성 검증

전체 UI를 만들기 전에 작은 실험 코드로 공통 작업 데이터를 유지한 계정 인증 교체와 실행 제어가 실제로 가능한지 확인한다.

구현·조사 항목:

- Codex 로컬 데이터에서 계정 인증에만 해당하는 항목과 계속 공유해야 하는 작업 데이터·설정의 경계를 찾는다.
- 인증 관련 항목을 불투명한 데이터로 백업하고 교체하는 것만으로 계정이 전환되는지 확인한다.
- 인증 데이터의 내용을 표시하거나 기록하지 않고 현재 적용된 인증 상태가 어느 프로필과 일치하는지 확인할 방법을 찾는다.
- 두 개의 시험 프로필에서 각각 로그인한 뒤 동일한 Codex 로컬 데이터 폴더를 사용해 계정만 번갈아 전환한다.
- 전환 전후에 기존 대화, 작업 기록, 프로젝트 연결, 설정 등 공통 작업 상태가 유지되는지 확인한다.
- 인증 교체 도중 실패했을 때 이전 인증 상태를 원래대로 복구할 수 있는지 확인한다.
- 프로필 전환 전후에 브라우저의 ChatGPT 로그인과 열어 둔 채팅이 유지되는지 확인한다.
- 실행 중인 Codex를 식별하고 정상 종료를 요청할 수 있는지 확인한다.
- 정상 종료가 되지 않을 때 강제 종료가 필요한 조건과 사용자 경고 내용을 확인한다.
- 지원하는 설치 유형에서 Codex 실행 파일을 자동 탐지할 수 있는지 확인한다.
- 경로, 로그, 오류 메시지에 인증 정보가 노출되지 않는지 확인한다.

완료 조건:

- 인증 관련 항목만 교체해 두 계정을 정확히 전환할 수 있다.
- 반복 전환 시 매번 다시 로그인하지 않아도 된다.
- 계정을 전환해도 동일한 Codex 작업 데이터와 설정을 계속 사용할 수 있다.
- 인증 교체 실패 시 전환 전 계정 상태로 복구된다.
- ChatGPT 웹 세션에 영향을 주지 않는다.
- Codex 설치 경로 자동 탐지, 실행, 감지, 정상 종료의 구현 방법을 재현 가능한 형태로 정리한다.

위 조건 중 인증 정보만의 안전한 교체, 공통 작업 데이터 보존 또는 웹 세션 보호가 불가능하면 본 구현을 중단하고, 지원 가능한 범위와 대안을 먼저 다시 결정한다.

### 15.3 1단계: 솔루션 뼈대와 책임 경계 구성

기술 검증 결과를 바탕으로 WPF 솔루션과 테스트 프로젝트를 만든다. 구체적인 프로젝트 수는 불필요하게 늘리지 않되 다음 책임은 분리한다.

- 핵심 규칙: 프로필 이름, 상태, 삭제 가능 여부처럼 UI나 파일 형식과 무관한 규칙
- 유스케이스: 프로필 목록 조회, 생성 및 로그인, 실행, 전환, 삭제의 작업 순서
- 외부 연동: 프로필 저장, 인증 상태 백업·교체·복구, Codex 설치 경로 탐지와 프로세스 실행·종료·감지
- WPF 화면: 사용자 입력, 상태 표시, 확인 대화상자, 유스케이스 호출
- 실행 진입점: 의존성 연결과 앱 시작

의존성은 WPF와 외부 연동 코드가 핵심 규칙과 유스케이스를 사용하도록 안쪽으로만 향하게 한다. 핵심 규칙은 WPF, 파일 시스템, 프로세스 API를 직접 참조하지 않는다.

완료 조건:

- 빈 앱이 빌드되고 실행된다.
- 핵심 규칙과 유스케이스 테스트가 WPF나 실제 Codex 없이 실행된다.
- 프로젝트 간 참조 방향이 위 책임 경계를 위반하지 않는다.

### 15.4 2단계: 프로필 모델과 로컬 저장 구현

프로필 목록을 빠르게 읽고 안전하게 저장하는 기반을 만든다.

구현 항목:

- 표시 이름과 별개인 내부 프로필 ID를 사용한다.
- 빈 이름, 공백 이름, 중복 이름을 거부한다.
- 프로필 메타데이터와 계정별 인증 상태 사본을 앱 전용 폴더에 저장한다.
- 공통 Codex 작업 데이터는 프로필 저장소로 복사하거나 프로필별로 나누지 않는다.
- 인증 상태 사본은 현재 Windows 사용자만 접근할 수 있도록 운영체제의 데이터 보호 기능과 파일 권한을 적용한다.
- 앱이 소유한 프로필 저장소 안에서만 생성과 삭제가 가능하도록 경로를 검증한다.
- 저장 도중 앱이 종료되어도 기존 목록이 손상되지 않도록 임시 파일 후 교체 방식 등 안전한 저장 방법을 사용한다.
- 손상되거나 읽을 수 없는 데이터는 앱 전체 중단 대신 이해 가능한 오류로 표시한다.

자동 테스트:

- 정상 프로필 생성과 목록 조회
- 빈 이름과 중복 이름 거부
- 저장 후 다시 실행했을 때 같은 목록 복원
- 손상된 저장 데이터 처리
- 앱 관리 범위 밖의 경로 삭제 차단
- 한 프로필의 저장·삭제가 공통 작업 데이터나 다른 프로필에 영향을 주지 않음

완료 조건:

- Codex를 실행하지 않고도 프로필 생성, 조회, 검증, 저장을 자동 테스트로 확인할 수 있다.

### 15.5 3단계: 최초 프로필 생성 및 로그인 흐름 구현

`새 프로필 추가` 화면과 기존 Codex 로그인 화면을 연결한다.

구현 항목:

- 프로필 이름 입력, 입력 오류, 취소, `생성 및 로그인` 동작을 구현한다.
- 현재 인증 상태가 있다면 복구용으로 안전하게 백업한 뒤 인증 관련 항목만 로그인 전 상태로 준비한다.
- 공통 Codex 데이터 폴더를 그대로 사용해 Codex 로그인 화면을 실행한다.
- 로그인 성공 여부를 인증 정보의 내용을 해석하지 않는 방식으로 판단하고, 생성된 인증 상태를 프로필에 저장한다.
- 로그인 취소 또는 실패 시 불완전한 프로필을 제거하고 기존 인증 상태를 복구한다.
- 생성 작업 중에는 같은 작업을 중복 실행하지 못하게 한다.

자동 테스트:

- 유효한 이름으로 생성 작업 시작
- 중복 이름과 빈 이름 차단
- 로그인 성공 시 사용 가능한 프로필로 저장
- 로그인 취소·실패 시 기존 인증 상태와 활성 프로필 복구
- 새 프로필 생성 전후 공통 작업 데이터 유지

수동 확인:

- 실제 Codex 로그인 완료 후 프로필이 `준비됨`으로 표시된다.
- 기존 Codex 작업 내역과 설정을 새 계정에서도 이어서 사용할 수 있다.
- 브라우저의 기존 ChatGPT 로그인과 채팅이 유지된다.

### 15.6 4단계: 프로필 실행과 현재 상태 표시 구현

Codex가 꺼져 있을 때 선택한 프로필로 실행하고 실제 상태를 화면에 반영한다.

구현 항목:

- 앱 시작 시 저장된 프로필 목록을 즉시 표시한다.
- Windows의 알려진 설치 위치와 등록 정보를 이용해 Codex 실행 파일을 자동 탐지한다.
- Codex 프로세스가 없으면 프로필별 `실행` 동작을 제공한다.
- 선택한 프로필의 인증 상태만 공통 Codex 데이터 위치에 적용한 뒤 Codex를 실행한다.
- 실행 성공을 확인한 뒤에만 해당 프로필을 활성 상태로 표시한다.
- 앱 밖에서 Codex가 종료된 경우 화면 상태를 갱신한다.
- 이미 다른 프로필이 실행 중이면 두 번째 Codex를 동시에 실행하지 않는다.

자동 테스트:

- Codex 미실행 상태에서 선택한 프로필 실행
- Codex 설치 경로 자동 탐지 성공과 실패
- 실행 실패 시 활성 상태를 변경하지 않음
- 다른 프로필 실행 중 중복 실행 차단
- 실제 프로세스·인증 상태와 저장된 표시 상태가 다를 때 실제 상태를 우선함

완료 조건:

- 사용자가 한 번의 `실행` 클릭으로 선택한 프로필의 Codex를 열 수 있다.
- 현재 활성 프로필과 실패 상태를 즉시 구분할 수 있다.

### 15.7 5단계: 안전한 프로필 전환 구현

전환은 하나의 유스케이스에서 순서대로 처리한다. 공통 작업 데이터는 건드리지 않으며, 인증 교체 중 실패하면 전환 전 인증 상태를 복구한다.

전환 순서:

1. 현재 실행 상태와 대상 프로필을 확인한다.
2. 중복 전환 입력을 잠그고 `전환 중` 상태를 표시한다.
3. 실행 중인 Codex에 정상 종료를 요청한다.
4. 제한 시간 안에 종료되지 않으면 작업 손실 가능성과 강제 종료 선택지를 표시한다.
5. 종료가 확인된 뒤 현재 인증 상태의 복구용 사본을 만든다.
6. 공통 작업 데이터는 그대로 두고 인증 관련 항목만 대상 프로필의 인증 상태로 교체한다.
7. 대상 프로필로 Codex를 실행한다.
8. 새 프로세스와 대상 계정의 적용 성공을 확인한 뒤 활성 프로필 표시를 변경한다.
9. 인증 교체 또는 실행이 실패하면 이전 인증 상태를 복구하고 실패 원인과 다음 행동을 표시한다.

자동 테스트:

- 정상 종료 후 대상 프로필 실행
- 정상 종료 실패 시 사용자 확인 전 강제 종료하지 않음
- 사용자가 취소하면 새 프로필을 실행하지 않음
- 강제 종료 동의 후 전환 재개
- 인증 교체 실패 시 이전 인증 상태 복구
- 대상 프로필 실행 실패 시 이전 인증 상태를 복구하고 성공으로 표시하지 않음
- 전환 전후 공통 작업 데이터의 파일과 설정을 변경하지 않음
- 전환 중 실행·전환·삭제 중복 조작 차단

수동 확인:

- 사용자는 한 번의 `전환` 클릭으로 일반적인 전환을 완료한다.
- 전환 전후 두 프로필의 로그인 상태, 공통 Codex 작업 상태와 ChatGPT 웹 세션이 유지된다.

### 15.8 6단계: 프로필 삭제 구현

삭제는 프로필 메타데이터와 앱이 저장한 해당 계정의 인증 상태 사본만 제거한다. 공통 Codex 작업 데이터와 설정은 제거하지 않는다.

구현 항목:

- 삭제 대상 이름, 저장된 인증 정보만 삭제된다는 점, 공통 작업 데이터는 유지된다는 점과 복구 불가능 여부를 확인 대화상자에 표시한다.
- 활성 프로필은 바로 삭제하지 않고 먼저 Codex 종료와 현재 적용된 인증 정보 제거가 필요한 이유 및 다음 행동을 안내한다.
- 삭제 대상 경로가 앱이 관리하는 정확한 프로필 저장소인지 다시 검증한다.
- 파일 삭제에 실패하면 목록에서 제거하지 않고 실패 원인과 재시도 방법을 표시한다.
- 삭제가 모두 성공한 뒤에만 프로필 목록에서 제거한다.

자동 테스트:

- 사용자 취소 시 아무 데이터도 삭제하지 않음
- 비활성 프로필의 메타데이터와 저장된 인증 상태만 삭제
- 활성 프로필 삭제 차단 또는 종료 흐름 연결
- 일부 파일 삭제 실패 시 프로필 목록 유지
- 다른 프로필, 공통 Codex 작업 데이터와 앱 관리 범위 밖의 파일을 삭제하지 않음

### 15.9 7단계: WPF 화면 상태와 사용성 마무리

각 기능을 메인 화면에서 일관된 상태로 연결한다.

구현 항목:

- 프로필 목록, 활성 강조, `실행`·`전환`, 관리 메뉴, `새 프로필 추가`를 배치한다.
- `준비됨`, `로그인 필요`, `실행 중`, `전환 중`, `확인 필요`, `실패` 상태를 구분한다.
- 긴 작업은 비동기로 실행해 화면이 멈추지 않게 하고 진행 중에는 충돌하는 조작만 비활성화한다.
- 오류 메시지에는 민감한 경로나 인증 데이터를 제외하고 원인과 다음 행동을 함께 표시한다.
- 키보드 이동, 포커스, 읽기 쉬운 대비와 같은 기본 접근성을 확인한다.

완료 조건:

- 일반적인 실행과 전환을 1~2번의 클릭으로 완료한다.
- 진행 중, 성공, 실패, 사용자 확인 필요 상태를 혼동하지 않는다.

### 15.10 8단계: 통합 검증과 회귀 테스트

실제 Codex를 사용하는 수동 시나리오와 외부 연동을 대체한 자동 테스트를 함께 실행한다.

필수 시나리오:

1. 빈 상태에서 첫 프로필 생성 및 로그인
2. 두 번째 프로필 생성 및 로그인
3. 각 프로필로 반복 실행 및 전환
4. 계정 전환 후에도 기존 Codex 작업 내역과 설정 유지
5. 인증 교체 실패 후 이전 계정으로 복구
6. Codex 정상 종료 실패 후 취소
7. Codex 정상 종료 실패 후 강제 종료 및 전환
8. 비활성 프로필 삭제 후 공통 작업 데이터 유지
9. 활성 프로필 삭제 시 안내 확인
10. 앱 재실행 후 프로필 목록과 실제 활성 상태 복원
11. Codex 설치 경로 자동 탐지 성공과 실패 안내
12. 손상된 설정 파일과 접근 권한 오류 처리
13. 모든 과정에서 ChatGPT 웹 로그인과 열린 채팅 유지

보안 확인:

- 소스 코드, 설정, 테스트 데이터, 로그에 인증 토큰이나 비밀번호가 없다.
- 오류 메시지와 진단 로그에 민감한 파일 내용이 없다.
- 저장된 인증 상태는 현재 Windows 사용자 이외에는 접근하기 어렵게 보호된다.
- 교체와 삭제 기능이 인증 관련 항목 및 앱 관리 폴더 밖으로 벗어나지 않는다.
- 공통 Codex 작업 데이터의 내용이 프로필 저장소에 불필요하게 복제되지 않는다.

완료 조건:

- 전체 자동 테스트와 Release 빌드가 통과한다.
- 12절의 성공 기준을 실제 환경에서 모두 확인한다.
- 알려진 제한사항과 재현 방법을 문서에 남긴다.

### 15.11 9단계: 배포용 실행 파일 준비

모든 테스트가 통과한 뒤 Windows x64용 단일 실행 파일을 만든다.

절차:

1. 전체 테스트를 실행한다.
2. Bootstrapper 프로젝트를 Release, `win-x64`, self-contained, single-file로 게시한다.
3. 게시된 실행 파일을 저장소 루트에 `codexUIUIUI.exe`라는 이름으로 복사한다.
4. 새 Windows 사용자 환경에서 실행, 프로필 생성, 전환, 삭제의 기본 동작을 최종 확인한다.

테스트가 실패하면 실행 파일을 준비하지 않고 실패 원인을 먼저 해결한다.
