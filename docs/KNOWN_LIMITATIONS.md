# 알려진 제한사항

## Codex 정상 종료

2026년 6월 18일 시험한 Microsoft Store형 Codex `26.611.8604.0`은
일반 창 닫기 요청에 10초 안에 종료되지 않을 수 있다.

현재 프로필 생성 로그인 흐름은 다음 순서를 사용한다.

1. Codex에 정상 종료를 요청한다.
2. 최대 10초 동안 종료를 기다린다.
3. 종료되지 않으면 작업 손실 가능성을 사용자에게 알린다.
4. 사용자가 명시적으로 동의한 경우에만 프로세스 트리를 강제 종료한다.

강제 종료는 임시방편이다. 저장되지 않은 입력이나 Codex가 기록 중인
상태가 손실될 수 있다.

프로필 전환에서는 사용자가 직접 `전환` 버튼을 누른 것을 전환 작업과
필요 시 강제 종료까지 승인한 입력으로 본다. 따라서 전환 흐름은 정상
종료 실패 시 추가 확인창 없이 강제 종료 후 전환을 계속한다. 최초 로그인
및 복구 흐름은 아직 별도 확인을 유지한다.

추후 Codex가 제공하는 신뢰할 수 있는 정상 종료 방식 또는 공식 종료
연동 지점을 확인하면 강제 종료 분기를 교체해야 한다.

## 3단계 기능 범위

- 새 프로필의 로그인 상태를 저장한 뒤 기존 인증 상태를 복구한다.
- 새 프로필로 Codex를 다시 실행하는 기능은 4단계에서 구현한다.
- 파일 인증 저장을 확인할 수 없는 환경이나 keyring 설정에서는 인증
  파일을 변경하지 않고 프로필 생성을 중단한다.

## 4단계 활성 프로필 판정

- 활성 프로필은 Codex 프로세스가 실행 중이고 현재 `auth.json`이 저장된
  인증 사본 하나와 정확히 일치할 때만 표시한다.
- Codex가 실행 중 인증을 갱신하거나 앱 밖에서 다른 계정으로 로그인하면
  저장 사본과 달라질 수 있다. 이 경우 마지막 실행 기록을 추측하지 않고
  `프로필 확인 불가`로 표시한다.
- 현재 인증을 마지막 실행 프로필에 자동 저장하지 않는다. 토큰 갱신과
  다른 계정 로그인을 인증 내용을 해석하지 않고 구별할 수 없기 때문이다.
- 프로필 전환과 실행 중 갱신된 인증 사본을 안전하게 보존하는 정책은
  5단계에서 다룬다.
